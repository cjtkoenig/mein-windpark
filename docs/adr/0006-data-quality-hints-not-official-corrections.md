# Data quality hints, not official corrections

WindKlar will treat the report flow as a Datenhinweis workflow for collecting structured hints about missing or incorrect wind installations and master data, not as an official MaStR correction mechanism. In the MVP, data hints are stored locally in SQLDelight with review/export state and are designed to be exportable later, but no backend submission or account-based workflow is required. This preserves the citizen-participation and data-quality value while avoiding the false promise that the app can directly alter public register data.
