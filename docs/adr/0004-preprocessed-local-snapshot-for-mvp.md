# Preprocessed local snapshot for MVP data

WindKlar will use preprocessed local MaStR source data for the MVP instead of fetching turbine master data from a live API inside the app. The pipeline may produce JSON snapshots as intermediate artifacts, but the app runtime ships a preseeded SQLite source database containing individual wind installations, precomputed wind park aggregates, summaries, search entries and snapshot metadata. Local user data lives in a separate SQLite database so source-data updates can replace only the source database. This keeps the dataset source-backed while reducing demo risk from network failures, rate limits, slow imports and runtime mapping complexity; later releases can add refresh or delta-import behavior.
